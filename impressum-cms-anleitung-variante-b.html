Impressum via CMS — Variante B: StaticPage-System

Schritt-für-Schritt-Anleitung: Impressum als STATIC-Page im bestehenden Pages-System pflegen

Übersicht

Variante B nutzt das bereits vorhandene Pages-System (Page-Entity mit HTML-Editor). Das Impressum wird als STATIC-Page angelegt und über den Slug impressum öffentlich ausgeliefert. Kein Backend-Code muss geändert werden — nur Frontend-Routing und eine neue Renderer-Komponente.

1

StaticPageRenderer-Komponente erstellen

Eine neue Komponente, die per Slug die veröffentlichte Page vom API-Endpunkt GET /api/pages/{slug} lädt und das HTML rendert.

Neu: src/pages/StaticPageRenderer.jsx
import React, { useState, useEffect } from 'react'; import { useParams } from 'react-router-dom'; import { Footer } from '../components/Footer'; import { Navbar } from '../components/Navbar'; export const StaticPageRenderer = () => { const { slug } = useParams(); const [page, setPage] = useState(null); const [error, setError] = useState(false); useEffect(() => { fetch(`/api/pages/${slug}`) .then(res => { if (!res.ok) throw new Error(); return res.json(); }) .then(data => setPage(data)) .catch(() => setError(true)); }, [slug]); if (error) return (<div>Seite nicht gefunden</div>); if (!page) return (<div>Laden...</div>); return ( <> <Navbar /> <main className="static-page-content"> <div dangerouslySetInnerHTML={{ __html: page.content }} /> </main> <Footer /> </> ); };
2

Route in App.jsx hinzufügen

Die bisherige hardcodierte /impressum-Route durch den dynamischen Renderer ersetzen. Zusätzlich eine Catch-all-Route für weitere Seiten (Datenschutz, AGB) einrichten.

Ändern: src/App.jsx
// Bisherig (entfernen): <Route path="/impressum" element={<ImpressumPage />} /> // Neu (hinzufügen): <Route path="/impressum" element={<StaticPageRenderer slug="impressum" />} /> <Route path="/datenschutz" element={<StaticPageRenderer slug="datenschutz" />} /> <Route path="/seite/:slug" element={<StaticPageRenderer />} />
Die festen Routen /impressum und /datenschutz bleiben für SEO erhalten. Die generische Route /seite/:slug fängt alle weiteren Seiten auf.
3

StaticPageRenderer um festen Slug erweitern

Damit sowohl /impressum (mit festem Slug-Prop) als auch /seite/:slug (mit URL-Parameter) funktionieren:

export const StaticPageRenderer = ({ slug: fixedSlug }) => { const { slug: paramSlug } = useParams(); const slug = fixedSlug || paramSlug; // ... Rest wie in Schritt 1 };
4

CSS für die gerenderten Seiten

Minimale Styles, damit die HTML-Inhalte im ProuDig-Design dargestellt werden. Die CSS-Klassen aus dem StaticPageEditor (section, container, grid, card) greifen bereits.

Ändern: src/App.css
/* Static Page Renderer */ .static-page-content { padding-top: 100px; /* Navbar-Höhe */ min-height: 60vh; } .static-page-content .section { padding: 4rem 0; } .static-page-content .container { max-width: 900px; margin: 0 auto; padding: 0 1.5rem; } .static-page-content h3 { margin: 1.5rem 0 0.5rem; color: var(--c-text); } .static-page-content p { line-height: 1.7; color: var(--c-text-muted); }
5

Impressum-Inhalt im Admin anlegen

Im Admin-Panel unter Seiten → Neue Seite eine neue STATIC-Page erstellen:

Titel: Impressum
Slug: impressum

Den bisherigen Impressum-Inhalt als HTML eintragen — der StaticPageEditor stellt eine CSS-Klassen-Referenz und Vorlagen bereit. Beispiel:

<section class="section"> <div class="container"> <div class="section-header"> <span class="section-tag">RECHTLICHES</span> <h2 class="section-title">Impressum</h2> </div> <h3>Angaben gemäß § 5 TMG</h3> <p>ProuDig GmbH<br> Musterstraße 123<br> 70173 Stuttgart</p> <!-- Weitere Abschnitte ... --> </div> </section>
6

Veröffentlichen und testen

Im Admin-Editor auf „Veröffentlichen” klicken. Danach:

1. localhost:5173/impressum öffnen — die Seite sollte den CMS-Inhalt anzeigen
2. Im Admin den Text ändern und erneut veröffentlichen
3. Seite im Browser neu laden — Änderung sofort sichtbar, kein Deployment nötig

7

Optional: Navbar-Links zu weiteren CMS-Seiten

Im Footer oder in der Navbar können Links zu /datenschutz, /agb etc. gesetzt werden. Sobald eine STATIC-Page mit dem passenden Slug veröffentlicht wird, ist die Seite live.

Die alte Impressum.jsx-Komponente und ImpressumPage.jsx können nach erfolgreicher Migration gelöscht werden.

Zusammenfassung

Geänderte Dateien: App.jsx (Routing), App.css (Styles)
Neue Dateien: StaticPageRenderer.jsx
Gelöschte Dateien: ImpressumPage.jsx, Impressum.jsx (nach Migration)
Backend: Keine Änderungen — API-Endpunkt GET /api/pages/{slug} existiert bereits
Ergebnis: Impressum, Datenschutz, AGB und beliebige weitere Seiten können vollständig aus dem CMS gepflegt werden — ohne Code-Änderung oder Deployment.
